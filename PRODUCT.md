# 最终产品方案

更新日期：2026-07-24

文档规则：

- “已确认”的内容是后续产品和Demo设计的唯一依据。
- 现有网页只用于验证交互，不能反过来定义产品。

## 0. 已确认的产品框架

产品由“感受”“创作”和“音乐库”三部分组成：

- **感受**：儿童从音乐库选择一首已经审核的音乐，用“画旋律、身体演奏、唱唱名”三种方式感受同一首作品。三种方式是并列按钮，儿童一次选择一种，也可以自由切换。
- **创作**：儿童用感觉贴纸、律动贴纸和动物贴纸编排音乐。声音均为提前制作并审核的素材，现场只做实时混音；一期以编排为主，不做自由作曲。
- **音乐库**：既是感受内容的来源，也是教师备课后台。它包含内置审核歌曲、教师上传并确认分析结果的歌曲，以及儿童完成的音乐作品。

感受和创作是两个独立入口。儿童可以先感受再创作，也可以跳过感受直接创作，不设置强制学习顺序。

首页只保留两个主要图形入口：戴耳机的动物代表“感受”，创作中的动物代表“创作”。不使用“今天想怎么玩音乐”等说明性标题。教师备课后台只作为角落里的小入口。

一期不设置独立投屏模式或投屏页面。课堂需要大屏时，直接显示主产品当前的感受页面。

### 不得误解的旋律曲线规则

> 旋律曲线的基本单位是“小节”：每个小节对应一张独立的曲线卡。它不是贯穿整首歌曲的一条连续彩色线，也不是先按整首歌或整句生成一条长线后再播放。

播放音乐时，系统根据小节起止时间依次呈现对应的曲线卡，引导儿童用手在空中跟随曲线移动。切换到下一小节时，画面同步切换到下一张曲线卡。相同旋律再次出现时，重复显示相同曲线；休止小节仍保留自己的空白时间卡。

曲线表达的是该小节中儿童能够听见的整体旋律轮廓，包括高低走向、持续长短、重复和停顿。它用于身体跟随和听觉感受，不是五线谱，也不要求显示每一个准确音符。

后台分析结果必须以小节为单位保存：

- 小节编号及开始、结束时间。
- 该小节对应的曲线形状或曲线模板编号。
- 曲线在画面中的高低位置和持续长度。
- 是否与其他小节重复。
- 是否为休止。

主产品中的功能名称使用“画旋律”。入口图案应表现多张分小节的曲线卡，不能使用一条贯穿整首歌曲的波形或长曲线。

### 不得误解的身体演奏规则

> 身体演奏同样以“小节”为显示单位，但一个小节内部再按“拍”排列动作：每一拍对应一个动作格。

例如一首四拍子的音乐，当前小节显示四个从左到右排列的动作格，分别对应第1、2、3、4拍。四个动作可以不同，也可以重复，例如“拍腿—拍手—拍胳膊—拍胳膊”或“拍桌—拍桌—拍手—拍手”。

播放时遵守以下规则：

- 当前小节开始前，完整显示该小节的全部动作格，让儿童知道接下来要做什么。
- 音乐播放后，系统按拍从左到右依次高亮动作格。
- 每到一拍，儿童完成该格对应的动作。
- 到达小节线时，整组动作格切换为下一小节的动作。
- 同一个动作可以在相邻拍、同一小节的不同拍或不同小节中重复。
- 休止也作为一个动作格，使用“停”图案，儿童在该拍不做动作。
- 一期只做到每拍一个主要动作，不拆成更细的半拍或四分之一拍动作，避免跟随难度过高。

后台身体演奏数据也必须以小节为单位保存：

- 小节编号及开始、结束时间。
- 该小节的拍号和拍数。
- 每一拍对应的动作库编号。
- 哪些拍使用重复动作。
- 哪一拍是重拍、弱拍或休止。

教师修改时不是重画整段动作，而是在当前小节中逐拍替换动作格；所有替换仍然只能来自审核后的动作库。

## 1. 产品定位

面向6—12岁儿童，尤其是乐器和专业教师资源不足的场景。

产品不是专业作曲软件，也不是让AI替儿童写歌。它提供两条独立但相连的路径：

1. **感受**：帮助儿童听懂、看见、唱出并用身体参与真实音乐。
2. **创作**：帮助儿童通过贴纸选择和编排完成自己的音乐作品。

核心原则：

> 儿童负责感受、选择和表达；AI负责分析、示范、连接和润色。

## 2. 感受

### 课堂入口

进入“感受”后，系统随机呈现一首已经审核的音乐。儿童可以直接开始，也可以点击“更换歌曲”进入审核歌曲页自由选择；不设置上一首、下一首按钮。

同一首音乐提供三个并列功能按钮：

- **画旋律**：按小节显示独立曲线，引导儿童用手跟随。
- **身体演奏**：按音乐拍点、重音和结构显示动作。
- **唱唱名**：跟随小兔用do、re、mi等唱名演唱。

儿童一次进入一种感受方式，可以随时返回切换。每首作品预先完成AI分析和教师审核，使用时不临时生成未经检查的教学内容。

播放、暂停和速度调整是“感受”播放器的基础控制，不作为额外学习模式。调整速度时，旋律曲线、逐拍动作高亮和唱名提示必须同步变化。

### 课堂微循环

每次只学习一个主要音乐现象，使用1—3分钟完成：

> 听一小段 → 发现一个特点 → 用身体体验 → 跟随屏幕完成 → 哼唱或说出来 → 换一种方式表现

可用的身体交互有两类：

- **旋律线条**：表现音高走向、乐句、长短和强弱。
- **身体打击动作**：表现稳定拍、节奏型、重拍和停顿。

身体动作以小节为一组显示，每拍一个动作格。轨道、倒数、灯光、高亮和口令只负责提示“什么时候做”，不是第三种学习内容。

### 屏幕的角色

屏幕是全班共同的领演者：

- 先示范动作，再给四拍准备。
- 提示与音乐重拍、乐句和段落同步。
- 教师可以选择“重复、简单一点、换动作”。
- 不使用摄像头，不判断动作正确与否，不评分。

屏幕不能替代教师对动作安全、儿童状态和开放回答的现场判断。

### AI动作库

动作库用于把AI分析出的音乐特点转换成儿童可以直接跟随的动作。AI只能选择、排列和重复库内动作，不能临时发明新动作。

所有动作默认站在原地完成，并遵守以下规则：

- 每个动作格只包含一个动作，一句口令只对应当前拍的动作。
- 使用“拍、跺、画、按、吹、停”等具体词，不使用“下沉、重量、能量、身体中心”等抽象描述。
- 屏幕动物先示范当前小节的动作顺序，再给起拍准备。
- 初次体验优先使用少量动作类型并适当重复；熟悉后可以增加变化，但始终保持每拍一个动作。
- 孩子站在自己的位置，不跑、不跳、不后退、不转圈。
- 不使用摄像头，不判断孩子是否完成，也不评分。
- 重拍通过动作位置和音色区分，不要求孩子加大力气。
- 吹奏动作只是假装吹，不要求深吸气或用力呼气。
- 不方便站立的孩子可以使用同一动作的坐姿版本。

#### 基础动作

| 编号 | 动物 | 动作 | 屏幕口令 | 主要匹配内容 |
| --- | --- | --- | --- | --- |
| D1 | 小狗 | 拍手 | 拍拍手 | 稳定拍、节奏型、轻拍 |
| D2 | 小狗 | 拍桌 | 拍拍桌子 | 重拍、节拍开始 |
| D3 | 小狗 | 拍腿 | 拍拍腿 | 稳定拍、音色变化 |
| D4 | 小狗 | 拍胳膊 | 拍拍胳膊 | 节奏型、音色变化 |
| C1 | 小猫 | 双脚轻跺 | 跺跺脚 | 低音、重音、稳定拍 |
| C2 | 小猫 | 左右脚交替 | 左脚，右脚 | 交替低音、均匀进行 |
| R1 | 小兔 | 手向上画弧线 | 手向上画 | 旋律上行 |
| R2 | 小兔 | 手向下画弧线 | 手向下画 | 旋律下行 |
| R3 | 小兔 | 手向旁边画弧线 | 画一条长线 | 长音、完整乐句 |
| B1 | 小熊 | 双手一起向下按 | 双手一起按 | 和弦、重音、同时发声 |
| B2 | 小熊 | 左右手交替向下按 | 左手，右手 | 交替伴奏、分解和弦 |
| L1 | 小狮子 | 手从嘴边向前送一下 | 吹一下 | 短音、重音、短乐句 |
| L2 | 小狮子 | 手从嘴边向前伸并保持 | 吹长一点 | 长音、持续乐句 |
| U1 | 全体 | 双手放在腿上 | 停 | 休止、乐句结束、段落结束 |

拍胳膊只拍上臂外侧。拍桌和跺脚都使用轻动作，不以声音大小判断重拍。

#### 节拍组合

| 音乐特点 | 动作组合 |
| --- | --- |
| 二拍子：强、弱 | 每小节显示2个动作格，例如拍桌—拍手 |
| 三拍子：强、弱、弱 | 每小节显示3个动作格，例如拍桌—拍手—拍手 |
| 四拍子：强、弱、次强、弱 | 每小节显示4个动作格，例如拍桌—拍手—轻拍桌—拍手 |
| 重复节奏型 | 同一动作可以占据多个动作格，也可以整组跨小节重复 |
| 休止 | 对应拍显示“停”动作格，双手放在腿上 |

#### 单人与合作

同一个动作只改变参与关系，不另外设计复杂动作。

| 模式 | 可用关系 | 简单口令 |
| --- | --- | --- |
| 单人家庭 | 跟动物一起做、动物一句孩子一句 | 跟我一起；轮到你 |
| 两人合作 | 同时、轮流、问答 | 一起做；你一下我一下；你先我后 |
| 全班合作 | 分组同时、两组问答、逐组加入 | 第一组；第二组；大家一起 |

小狗可以在两人合作时使用轻轻击掌；其他动物保持原有动作，只改变同时、轮流或问答关系。

#### AI匹配顺序

1. 分析歌曲的稳定拍、速度、节拍、节奏型、重拍、低音、旋律走向、和弦、长短音、休止、乐句和段落。
2. 根据拍号划分小节，并确定每个小节包含几个动作格。
3. 根据每一拍的重音、节奏功能和音色特点，从动作库选择一个动作；动作允许重复。
4. 把一个小节内的动作按拍序排列，播放时逐拍高亮。
5. 家庭使用单人关系；教室优先使用同桌、分组或全班关系。
6. 在播放前检查口令是否直接、动作是否能站在原地完成、是否需要桌面或同伴。
7. 没有桌面时，用拍腿替代拍桌；没有同伴时，由屏幕动物承担合作角色。

教学真实作品时，动物只负责带领身体动作，教学乐器贴纸仍负责说明原作中的真实乐器。例如小提琴演奏旋律时，可以显示小提琴贴纸，并由小兔带领孩子画旋律线，但不能说成“小兔正在演奏小提琴”。

### 旋律曲线图形库

首版只生成旋律线，不生成和弦、低音或其他音乐图形。

旋律线放在画面中央，用左右位置表示小节内的时间、上下位置表示相对音高，由小兔带领。系统先按拍点和小节线切分音乐，再为每个小节生成一张独立曲线卡。

一张曲线卡可以由M1—M7中的基本形状组合而成，但在儿童看到的画面上，它仍然是该小节的一条完整曲线。M8使用空白时间卡和“停”的提示，不绘制曲线。AI只能使用审核后的基本形状和组合规则；所有分析结果必须经过教师确认后才能发布。

#### 按小节播放

核心播放方式是一次突出显示当前小节：

- 当前小节开始时，出现该小节对应的独立曲线卡。
- 播放光点和小兔沿当前曲线从左向右移动，儿童用手同步描画。
- 到达小节线时，当前曲线结束，画面切换到下一小节的曲线卡。
- 相同旋律在不同小节重复时，必须分别显示相同曲线，不能合并或省略。
- M8休止仍占用一个完整小节位置，保持空白并显示“停”。

例如《两只老虎》开头两个小节都是“1 2 3 1”，第一小节先显示一张对应曲线；进入第二小节后，再显示一次相同曲线。两次显示属于两个独立小节，不能画成一条跨越两个小节的长线。

#### 旋律线形状

| 编号 | 线条形状 | 表示内容 | 小兔动作口令 |
| --- | --- | --- | --- |
| M1 | 连续横线 | 同一音高的长音 | 画一条长线 |
| M2 | 分开的短横线 | 同一音高的多个短音 | 一下，一下 |
| M3 | 向上的弧线 | 旋律上行 | 手向上画 |
| M4 | 向下的弧线 | 旋律下行 | 手向下画 |
| M5 | 拱形线 | 先上行，再下行 | 先上，再下 |
| M6 | 凹形线 | 先下行，再上行 | 先下，再上 |
| M7 | 简单波浪线 | 旋律反复上行、下行 | 上，下，上，下 |
| M8 | 线条中断 | 休止或乐句之间的空隙 | 停 |

线条还可以使用以下变化，但同一活动最多强调一种：

- 线条越长，声音持续越久。
- 线条越高，音高越高；线条越低，音高越低。
- 播放光点沿线条移动，表示当前听到的位置。
- 每个小节的曲线独立呈现，不连接前后小节的首尾。
- 小节切换不代表休止，只有M8空白时间卡代表休止。
- 相同旋律小节再次出现时，必须复用相同的形状、位置和长度。

旋律线只表现儿童能够听见的整体轮廓，不要求显示每一个准确音符。音频分析不确定时，宁可使用较少、较大的线段，也不生成复杂曲线。

#### AI生成顺序

1. 有乐谱或音符资料时，优先按照资料生成旋律线。
2. 只有音频时，生成简化的旋律轮廓。
3. 根据拍点和节拍先划分小节，再提取每个小节的整体旋律轮廓。
4. 每个小节使用一张曲线卡；曲线卡可由少量基本形状组合。
5. 为每张曲线卡配置一个直接、适龄的跟随口令。
6. 播放时按小节依次出现，孩子跟随小兔画线。

#### 提供给AI的固定规则

动作库和图形库是AI生成课堂内容时必须同时读取的设计规范。AI不能跳过规范，也不能自己创造新的线条含义。

AI分析完音乐后，先确定每个小节的起止时间，再为每个小节选择一种或少量标准形状，组合成该小节唯一的一张曲线卡：

| AI分析结果 | 必须使用的图形 |
| --- | --- |
| 同一音高持续 | M1 连续横线 |
| 同一音高连续出现短音 | M2 分开的短横线 |
| 整体向上 | M3 向上的弧线 |
| 整体向下 | M4 向下的弧线 |
| 先上后下 | M5 拱形线 |
| 先下后上 | M6 凹形线 |
| 多次上下反复 | M7 简单波浪线 |
| 没有声音或乐句空隙 | M8 线条中断 |

每个小节的生成结果必须包括：

- 小节编号及开始、结束时间。
- 使用哪个标准图形。
- 基本形状在小节内部从什么时候开始、什么时候结束。
- 图形位于画面的高处、中间还是低处。
- 是否复用其他小节的曲线。
- 是否为休止小节。
- 对应哪只动物。
- 屏幕说什么简单口令。
- 使用单人、两人还是全班方式。

画面系统根据小节编号和曲线数据调用审核后的图形。不同小节的曲线不能连接成整首长线。分析结果不明确时，使用更简单的整体轮廓；无法确定旋律时，不显示未经教师确认的曲线。

### 教学乐器贴纸

原作中的乐器只画乐器本身，例如小提琴、古筝、长笛和琵琶，不增加动物角色。

教学乐器贴纸用于：

- 显示乐器真实名称。
- 连接音色、演奏动作和原作中的具体声音。
- 配合短音频、旋律线和身体模仿。

视觉规则已经确定：

- 教学乐器贴纸使用规则的长方形卡面，固定出现在课堂讲解区。
- 卡面只显示完整乐器、真实名称和当前聆听提示。
- 不显示拖动、添加、上场或生成作品的反馈。
- 五只动物使用有白边的彩色角色贴纸，出现在创作贴纸盒并可以拖动。

教学乐器贴纸不能进入四段时间轴、儿童作品或乐队合照。

例如学习《四季》时，小提琴贴纸配合原作片段和运弓动作出现；儿童可以学习长音、短音、快速反复和独奏问答，但小提琴不会成为创作音轨。

### 课程材料

课堂不只使用西方经典作品，也包括中国童谣、地方民歌、民族器乐和儿童熟悉的生活音乐。

达尔克罗兹、柯达伊、奥尔夫和铃木只作为设计依据，不做成四个课程入口：

- 达尔克罗兹：身体感受、主动聆听和即兴。
- 柯达伊：哼唱、内在听觉、音乐母语和循序学习。
- 奥尔夫：动作、语言、合作与创编。
- 铃木：重复聆听、模仿和渐进练习。

产品不宣称完整实施其中任何一种体系，也不宣称必然提高智力或学业成绩。

## 3. 音乐库与教师备课后台

音乐库是“感受”的统一内容来源，不是儿童首页上的第三个并列入口。儿童在“感受”中点击“更换歌曲”后进入审核歌曲页，从中自由选择。

音乐库包含三类内容：

- 产品内置并已审核的音乐。
- 教师上传、经AI分析并由教师确认后发布的音乐。
- 儿童完成并保存在当前设备中的音乐作品。

教师后台通过主产品角落里的小入口进入。教师可以同时上传音源和乐谱，也可以只上传音源：

> 上传音源与可选乐谱 → AI按小节分析旋律曲线、身体动作和唱名 → 教师预览与修改 → 教师确认发布 → 进入审核歌曲页

其中：

- 有乐谱时优先读取乐谱中的主旋律、节奏、拍号和小节信息，并与音源时间对齐。
- 乐谱可以是结构化文件，也可以是图片或PDF；图片和PDF需要先识别，识别结果必须由教师确认。
- 没有乐谱时，AI从音源中识别主旋律和小节；无法可靠识别时不能直接发布。
- 唱名根据最终确认的主旋律生成，并与原音源、小熊键盘和小节时间保持同步。
- 旋律分析必须按小节输出独立曲线卡及时间信息。教师可以替换曲线模板，调整高低位置和持续长度，但不能自由绘制曲线。
- 身体动作必须按小节和拍位输出；每拍对应一个动作格。动作只能从已审核动作库中选择，教师也只能逐拍在动作库内替换。
- AI分析结果只是候选内容，未经教师确认的音乐不能出现在儿童可见的审核歌曲页。
- 发布后的音乐在相应内容完成审核后，开放“画旋律、身体演奏、唱唱名”功能。
- 儿童创作完成后，也可以按同一规则转译成三种感受内容，再次用于个人体验或课堂活动。

## 4. 儿童音乐创作

### 主流程

> 选择感觉与律动 → 动物编排 → 实时混音 → 可选AI优化 → 生成旋律线与身体演奏

节奏热身移动到教师课堂端。儿童创作一期不录制拍手，也不现场生成新的节奏声音；律动贴纸直接选择提前制作好的节奏型。

感受和创作是两个独立入口，但共享“按小节画旋律”和“按小节、逐拍做身体演奏”两种表达方式。感受可以启发孩子进入创作，儿童作品也可以再次转译成个人体验或全班活动。

### 感觉贴纸

首版提供开心、安静、勇敢和想念四种音乐起点。

感觉贴纸选择提前制作好的旋律材料、调性色彩和视觉氛围，不判断儿童心理状态，也不把某种感觉固定成唯一的性格解释。

### 律动贴纸

- 律动贴纸选择提前制作好的节奏型、速度和重音方式。
- 儿童端使用“稳稳走、蹦蹦跳、摇一摇、向前冲”等身体化名称，不把律动简单等同于音乐流派。
- 感觉和律动共同确定一套音乐素材；两者选定后不在课堂现场生成新声音。
- 节奏热身保留在教师课堂端，不再要求儿童录制拍手。

### 五只动物乐队

| 动物 | 音乐职责 | 身体动作 |
| --- | --- | --- |
| 小狗 | 鼓和预设节奏型 | 拍手 |
| 小兔 | 使用do、re、mi等唱名演唱旋律 | 单手画弧线 |
| 小熊 | 使用键盘演奏主旋律和简单和声 | 双手向下按 |
| 小猫 | 贝斯和低音稳定感 | 轻轻跺脚 |
| 小狮子 | 小号短句、长短音、强弱和呼吸 | 手从嘴边向前送，假装吹奏 |

小熊和小兔使用同一份旋律乐谱：小熊提供稳定的键盘音高参考，小兔按相同音高和节奏演唱唱名。只有小熊时是器乐旋律，只有小兔时是唱名旋律，两者同时出现时形成键盘领奏与唱名演唱。小号只在乐句空隙、重音和结尾出现，与唱名形成问答或留白。

这五只动物是儿童创作的固定配置。其他教学乐器不能加入。

### 小兔唱名模式

一期只提供唱名演唱，不加入“啦啦唱”和歌词演唱。

- 使用do、re、mi、fa、sol、la、si等唱名。
- 小兔唱名和小熊键盘必须来自同一份旋律数据，不能分别自由生成。
- 主产品提供“听小兔唱、和小兔一起唱、轮到我唱”三个课堂状态。
- “轮到我唱”时小兔静音，小熊继续提供键盘音高参考。
- 不录制儿童声音，不判断音准，也不评分。

歌词演唱作为后续功能，需要在唱名模式、儿童跟唱和教师课堂验证稳定后再加入。

### 古诗演唱示例

古诗不做成独立产品入口，也不加入一期创作步骤。它只作为一件提前制作好的“灵感示例”，提示孩子：自己编排的音乐也可以用来唱古诗。

首个示例建议使用《静夜思》：

- 提前完成一套四段八小节的音乐编排，并人工确认古诗与旋律、节奏和呼吸位置匹配。
- 四个乐段分别对应四句诗，每段两小节。
- 完整示范版由小兔或示范人声唱古诗；伴奏版保留小熊键盘和其他动物，去掉小兔唱名，让孩子自己唱。
- 播放时一次突出当前诗句，可按节拍逐字提示。
- 示例可以放入音乐库，并提供“我也来编”的轻入口，直接返回现有贴纸创作。

该示例不代表任意古诗都能自动套入任意作品。产品不做AI实时古诗作曲、自动填词、儿童录音、音准判断或演唱评分。小兔在正式一期功能中仍只唱唱名，古诗演唱是孩子使用自己作品的一种开放玩法。

### 四段编排

- 整首作品分为四个乐段，每段两小节，共八小节。
- 小狗默认出现在四个乐段中，保证第一次试听就有稳定节奏；儿童可以移除。
- 动物可以出现在多个乐段，但同一乐段不能重复。
- 同时支持拖动和“先点动物、再点乐段”。
- 点击单段可试听，“从头听”依次播放四段。
- 编排页上方保持纸偶舞台，下方保持时间轴和贴纸盒。
- 动物只在乐段边界上下场；完整播放后形成乐队合照。
- 作品数据仍然只有八小节，完整体验默认连续播放两遍，整体控制在30—60秒。
- 第一遍用于完整聆听或观看示范，第二遍用于跟唱、画旋律或身体演奏；儿童可以随时暂停。

当前版本不加入速度、力度或音乐方法贴纸，避免时间轴过于复杂。

### 实时混音与轻量AI优化

每组“感觉＋律动”对应五条提前制作、长度相同、拍点对齐的动物循环分轨。四个乐段不需要分别制作音频，只记录每只动物在哪些乐段发声。

儿童完成编排后，系统立即生成“我的实时混音”：

- 五条分轨从同一拍点保持同步。
- 系统只在乐段边界打开或关闭相应动物音轨。
- 未选择的动物不发声。
- 原始版本不依赖联网或AI生成，必须能够立即播放。
- 作品播放器提供播放、暂停和速度调整；速度调整只是播放辅助，不改变孩子选择的律动贴纸和原始作品数据。

“AI优化版”是可选的轻量处理，不重新作曲，也不重新生成整首音乐。它只可以：

- 调整音量平衡、均衡、空间感和整体响度。
- 在乐段边界加入淡入淡出。
- 从经过审核的素材库中选择过渡和结束音。
- 降低循环拼接感，让四段连接得更自然。

AI优化不能改变动物出场时间、增加未选择的乐器、替换基础旋律或覆盖儿童的原始版本。优化失败时，实时混音版仍可正常播放、保存和进入课堂。

### AI作品转译

完成编排后，系统根据已知的音轨、拍点和动物出场数据，把儿童作品转化为可理解、可跟随的课堂内容；语言模型只负责生成适龄的提示和教师引导语。

AI可以：

- 按小节显示独立旋律曲线，并标明四段结构。
- 按小节和拍位生成拍手、拍腿、停顿等身体演奏。
- 生成适合全班跟随的提示和简短教师引导语。
- 提供作品标题和寄语建议。

AI不能：

- 改变动物出场。
- 改变儿童选择的感觉贴纸、律动贴纸或基础节奏型。
- 擅自加入儿童没有选择的乐器。
- 把自己的版本标成儿童原创。

儿童可以返回修改作品。AI分析失败时，原始编排仍能正常播放和保存。

### 音乐明信片

最终作品包含标题、情绪背景、动物乐队合照、播放按钮、作者标识和一句寄语。

“替作品说一句话”自然融入明信片制作：作品完成后，小兔询问“如果音乐会说话，它想说什么？”孩子可以选择一句示例，也可以说一个词或一句自己的话，由老师或家长帮助记录。产品不要求录音、不做语音识别，也不评价表达是否正确。

固定显示：

> 由我选择与编排 · AI提供音乐素材和混音帮助

一期作品只保存在当前设备，并在当前设备的音乐库中出现。不设置班级音乐库、公开广场、点赞、排行或陌生人评论。若后续提供对外分享，必须经过成年人确认，并只生成私密、有期限的链接。

### 一起演

“一起演”是连接创作与课堂的核心体验：

1. 先做一次简短动作示范。
2. 给四拍起拍。
3. 屏幕按动物出场提示动作。
4. 完整表演后共同挥手结束。

设备不录音、不录像、不识别动作，也不评分。

## 5. AI与技术分工

| 工作 | 负责方式 |
| --- | --- |
| 感觉、律动和五只动物的声音 | AI提前生成、人工审核的分轨素材 |
| 四段动物出场与即时播放 | 本地实时混音 |
| 音量、淡入淡出、过渡、结尾和整体响度 | 轻量AI混音与审核素材库 |
| 拍点、响度和音高曲线 | 本地或专业音频算法 |
| 音乐结构、乐器、情绪和教学语言解释 | 语言模型 |
| 旋律线、动作、口令和难度版本 | AI生成候选脚本 |
| 教学安全、音乐准确性和年龄适配 | 音乐教师或产品审核人员最终确认 |

音乐素材采用“提前生成、人工审核、稳定播放”。儿童完成编排后先得到本地实时混音；联网AI优化是可选增强，不阻塞创作和课堂。

## 6. 参赛Demo范围

Demo只验证：

> 普通教师能否把一首歌曲快速变成可用的音乐活动；孩子能否在体验之后完成作品；这件作品能否再次变成全班共同参与的课堂内容。

规划中的完整参赛Demo包括：

- 教师模式中的一首内置示范歌曲。
- 按小节画旋律、按小节逐拍身体演奏和课堂播放。
- 感觉贴纸与律动贴纸。
- 五只动物、四段时间轴和纸偶舞台。
- 原始实时混音与轻量AI优化对比。
- 儿童作品的旋律线与身体演奏转译。
- 音乐明信片、本地保存和分享确认作为可选支线。
- 无摄像头、无动作评分。

任意歌曲上传、正式课程库、麦克风拍手识别和教学乐器贴纸不进入当前三分钟Demo，作为下一阶段。

现有网页只是用于验证部分流程的交互原型，尚未完整落实本文件中的首页、感受入口、审核歌曲页、按小节画旋律和教师后台。后续修改Demo时以本文件为唯一产品依据，不能根据旧页面反推产品规则。

完整版及参赛Demo音乐素材由4种感觉、4种律动和5只动物组成，共16套音乐组合、80条对齐的循环分轨。四个乐段重复使用同一套分轨，只控制音轨开关；另外准备少量经过审核的过渡和结束素材。参赛Demo直接制作完整80条分轨，不采用20条缩减方案。正式素材未完成前，原型继续使用浏览器合成音。

## 7. 三分钟演示

1. **0:00—0:20**：说明非专业教师缺少音乐活动设计能力。
2. **0:20—0:55**：选择一首内置歌曲，展示AI生成的旋律线、身体演奏和教师引导语。
3. **0:55—1:20**：进入“感受”，选择一种模式完成一轮按小节跟随。
4. **1:20—1:35**：孩子选择一种感觉，进入贴纸创作。
5. **1:35—2:10**：把动物放入四段时间轴并试听作品。
6. **2:10—2:35**：把儿童作品转化为旋律线和身体演奏。
7. **2:35—3:00**：全班点击“一起演”，共同表演孩子创作的音乐。

演示结束语：

> AI先帮助老师把音乐变得可教，再帮助孩子把感受变成作品，最后让孩子的作品成为全班共同演奏的音乐。

## 8. 唱名方式

首版统一采用首调唱名：无论音乐是什么调，都把当前调的主音唱作do，再根据调内关系生成其他唱名。

首调唱名用于帮助儿童感受旋律级数关系和开口跟唱，不用于绝对音高训练。上传乐谱或从音源识别主旋律后，小兔唱名、小熊键盘和旋律曲线都必须使用同一份最终确认的旋律数据。
