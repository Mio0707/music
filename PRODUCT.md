# 最终产品方案

更新日期：2026-07-23

## 1. 产品定位

面向6—12岁儿童，尤其是乐器和专业教师资源不足的场景。

产品不是专业作曲软件，也不是让AI替儿童写歌。它提供两条独立但相连的路径：

1. **AI音乐课堂**：帮助儿童听懂、看见并用身体参与真实音乐。
2. **儿童音乐创作**：帮助儿童把拍手、情绪和音乐选择变成完整作品。

核心原则：

> 儿童负责感受、选择和表达；AI负责分析、示范、连接和润色。

## 2. AI音乐课堂

### 课堂入口

低龄儿童从一首具体作品和画面情境进入，不先从抽象术语进入。首批课程中的中国音乐与世界音乐数量各半。

每首作品预先完成AI分析和教师审核，课堂中不临时生成未经检查的教学内容。

### 课堂微循环

每次只学习一个主要音乐现象，使用1—3分钟完成：

> 听一小段 → 发现一个特点 → 用身体体验 → 跟随屏幕完成 → 哼唱或说出来 → 换一种方式表现

可用的身体交互有两类：

- **旋律线条**：表现音高走向、乐句、长短和强弱。
- **身体打击动作**：表现稳定拍、节奏型、重拍和停顿。

轨道、倒数、灯光和口令只负责提示“什么时候做”，不是第三种学习内容。

### 屏幕的角色

屏幕是全班共同的领演者：

- 先示范动作，再给四拍准备。
- 提示与音乐重拍、乐句和段落同步。
- 教师可以选择“重复、简单一点、换动作”。
- 不使用摄像头，不判断动作正确与否，不评分。

屏幕不能替代教师对动作安全、儿童状态和开放回答的现场判断。

### AI动作库

动作库用于把AI分析出的音乐特点转换成儿童可以直接跟随的动作。AI只能选择、排列和重复库内动作，不能临时发明新动作。

所有动作默认站在原地完成，并遵守以下规则：

- 一句口令只包含一个动作。
- 使用“拍、跺、画、按、吹、停”等具体词，不使用“下沉、重量、能量、身体中心”等抽象描述。
- 屏幕动物先示范，再给四拍准备。
- 第一次只使用一种动作，熟悉后组合两种，单次活动最多使用三种。
- 孩子站在自己的位置，不跑、不跳、不后退、不转圈。
- 不使用摄像头，不判断孩子是否完成，也不评分。
- 重拍通过动作位置和音色区分，不要求孩子加大力气。
- 吹奏动作只是假装吹，不要求深吸气或用力呼气。
- 不方便站立的孩子可以使用同一动作的坐姿版本。

#### 基础动作

| 编号 | 动物 | 动作 | 屏幕口令 | 主要匹配内容 |
| --- | --- | --- | --- | --- |
| D1 | 小狗 | 拍手 | 拍拍手 | 稳定拍、节奏型、轻拍 |
| D2 | 小狗 | 拍桌 | 拍拍桌子 | 重拍、节拍开始 |
| D3 | 小狗 | 拍腿 | 拍拍腿 | 稳定拍、音色变化 |
| D4 | 小狗 | 拍胳膊 | 拍拍胳膊 | 节奏型、音色变化 |
| C1 | 小猫 | 双脚轻跺 | 跺跺脚 | 低音、重音、稳定拍 |
| C2 | 小猫 | 左右脚交替 | 左脚，右脚 | 交替低音、均匀进行 |
| R1 | 小兔 | 手向上画弧线 | 手向上画 | 旋律上行 |
| R2 | 小兔 | 手向下画弧线 | 手向下画 | 旋律下行 |
| R3 | 小兔 | 手向旁边画弧线 | 画一条长线 | 长音、完整乐句 |
| B1 | 小熊 | 双手一起向下按 | 双手一起按 | 和弦、重音、同时发声 |
| B2 | 小熊 | 左右手交替向下按 | 左手，右手 | 交替伴奏、分解和弦 |
| L1 | 小狮子 | 手从嘴边向前送一下 | 吹一下 | 短音、重音、短乐句 |
| L2 | 小狮子 | 手从嘴边向前伸并保持 | 吹长一点 | 长音、持续乐句 |
| U1 | 全体 | 双手放在腿上 | 停 | 休止、乐句结束、段落结束 |

拍胳膊只拍上臂外侧。拍桌和跺脚都使用轻动作，不以声音大小判断重拍。

#### 节拍组合

| 音乐特点 | 动作组合 |
| --- | --- |
| 二拍子：强、弱 | 拍桌—拍手 |
| 三拍子：强、弱、弱 | 拍桌—拍手—拍手 |
| 四拍子：强、弱、次强、弱 | 拍桌—拍手—轻拍桌—拍手 |
| 重复节奏型 | 在拍手、拍桌、拍腿中选择两种或三种排列 |
| 休止 | 动作停止，双手放在腿上 |

#### 单人与合作

同一个动作只改变参与关系，不另外设计复杂动作。

| 模式 | 可用关系 | 简单口令 |
| --- | --- | --- |
| 单人家庭 | 跟动物一起做、动物一句孩子一句 | 跟我一起；轮到你 |
| 两人合作 | 同时、轮流、问答 | 一起做；你一下我一下；你先我后 |
| 全班合作 | 分组同时、两组问答、逐组加入 | 第一组；第二组；大家一起 |

小狗可以在两人合作时使用轻轻击掌；其他动物保持原有动作，只改变同时、轮流或问答关系。

#### AI匹配顺序

1. 分析歌曲的稳定拍、速度、节拍、节奏型、重拍、低音、旋律走向、和弦、长短音、休止、乐句和段落。
2. 每个活动只选择一个主要学习内容。
3. 根据主要内容选择一只动物和一个基础动作。
4. 家庭使用单人关系；教室优先使用同桌、分组或全班关系。
5. 先生成一种动作，必要时加入第二种对比动作；最多使用三种。
6. 在播放前检查口令是否直接、动作是否能站在原地完成、是否需要桌面或同伴。
7. 没有桌面时，用拍腿替代拍桌；没有同伴时，由屏幕动物承担合作角色。

教学真实作品时，动物只负责带领身体动作，教学乐器贴纸仍负责说明原作中的真实乐器。例如小提琴演奏旋律时，可以显示小提琴贴纸，并由小兔带领孩子画旋律线，但不能说成“小兔正在演奏小提琴”。

### 旋律线图形库

首版只生成旋律线，不生成和弦、低音或其他音乐图形。

旋律线放在画面上方，用左右位置表示时间、上下位置表示相对音高，由小兔带领。一个乐句可以拆成一至四个清楚的旋律小段，每个小段选择一张提前制作好的曲线图片。

M1—M7分别制作短、中、长三种图片样式。M8使用空白时间格和“停”的提示，不绘制曲线。AI只能选择图片、长度版本、摆放位置和出现时间，不能现场绘制或改变曲线形状。

#### 两小节显示窗口

屏幕每次同时显示两个小节，左右各一个时间格：

- 两个小节从播放前就同时显示，孩子可以提前看到下一小节。
- 当前小节使用正常颜色，下一小节降低亮度。
- 播放光点和小兔只在当前小节移动。
- 第一小节结束后，第二小节恢复正常颜色，第一小节停止动画。
- 第二小节结束后，整组画面切换到后面两个小节。
- 每个小节最多放置两个曲线图片，两小节合计最多四个。
- 相同旋律在两个小节中重复时，必须把同一曲线图片分别显示两次，不能合并或省略第二次。
- M8休止仍占用自己的时间位置，保持空白并显示“停”。

例如《两只老虎》开头两个小节都是“1 2 3 1”，画面左右各显示一张相同的M5拱形图片。小兔先走过左边的M5，再走过右边的M5。

#### 旋律线形状

| 编号 | 线条形状 | 表示内容 | 小兔动作口令 |
| --- | --- | --- | --- |
| M1 | 连续横线 | 同一音高的长音 | 画一条长线 |
| M2 | 分开的短横线 | 同一音高的多个短音 | 一下，一下 |
| M3 | 向上的弧线 | 旋律上行 | 手向上画 |
| M4 | 向下的弧线 | 旋律下行 | 手向下画 |
| M5 | 拱形线 | 先上行，再下行 | 先上，再下 |
| M6 | 凹形线 | 先下行，再上行 | 先下，再上 |
| M7 | 简单波浪线 | 旋律反复上行、下行 | 上，下，上，下 |
| M8 | 线条中断 | 休止或乐句之间的空隙 | 停 |

线条还可以使用以下变化，但同一活动最多强调一种：

- 线条越长，声音持续越久。
- 线条越高，音高越高；线条越低，音高越低。
- 播放光点沿线条移动，表示当前听到的位置。
- 每张曲线图片独立摆放，不要求前一张的尾端连接后一张的开头。
- 图片之间使用固定时间格分隔；普通分隔不代表休止，只有M8空白时间格代表休止。
- 相同旋律再次出现时，必须复用同一组形状、位置和长度。

旋律线只表现儿童能够听见的整体轮廓，不要求显示每一个准确音符。音频分析不确定时，宁可使用较少、较大的线段，也不生成复杂曲线。

#### AI生成顺序

1. 有乐谱或音符资料时，优先按照资料生成旋律线。
2. 只有音频时，生成简化的旋律轮廓。
3. 先划分乐句，再把每个乐句拆成一至四个清楚的旋律小段。
4. 每个小段选择一种固定形状，并对应一个直接口令。
5. 一个乐句最多连续使用四种基本形状。
6. 屏幕示范一次后，孩子跟随小兔画线。

#### 提供给AI的固定规则

动作库和图形库是AI生成课堂内容时必须同时读取的设计规范。AI不能跳过规范，也不能自己创造新的线条含义。

AI分析完音乐后，先把每个乐句拆成一至四个旋律小段，再给每个小段选择一个标准标签：

| AI分析结果 | 必须使用的图形 |
| --- | --- |
| 同一音高持续 | M1 连续横线 |
| 同一音高连续出现短音 | M2 分开的短横线 |
| 整体向上 | M3 向上的弧线 |
| 整体向下 | M4 向下的弧线 |
| 先上后下 | M5 拱形线 |
| 先下后上 | M6 凹形线 |
| 多次上下反复 | M7 简单波浪线 |
| 没有声音或乐句空隙 | M8 线条中断 |

每个旋律小段的生成结果必须包括：

- 使用哪个标准图形。
- 图形从什么时候开始、什么时候结束。
- 使用短、中、长哪个图片版本。
- 图形位于画面的高处、中间还是低处。
- 对应哪只动物。
- 屏幕说什么简单口令。
- 使用单人、两人还是全班方式。

画面系统根据编号调用提前制作好的图片模板。AI不能自由绘制、拉扯或修改曲线，也不需要连接相邻图片。分析结果不明确时，减少图片数量；无法确定旋律时，不显示未经确认的图片。

### 教学乐器贴纸

原作中的乐器只画乐器本身，例如小提琴、古筝、长笛和琵琶，不增加动物角色。

教学乐器贴纸用于：

- 显示乐器真实名称。
- 连接音色、演奏动作和原作中的具体声音。
- 配合短音频、旋律线和身体模仿。

视觉规则已经确定：

- 教学乐器贴纸使用规则的长方形卡面，固定出现在课堂讲解区。
- 卡面只显示完整乐器、真实名称和当前聆听提示。
- 不显示拖动、添加、上场或生成作品的反馈。
- 五只动物使用有白边的彩色角色贴纸，出现在创作贴纸盒并可以拖动。

教学乐器贴纸不能进入四段时间轴、儿童作品或乐队合照。

例如学习《四季》时，小提琴贴纸配合原作片段和运弓动作出现；儿童可以学习长音、短音、快速反复和独奏问答，但小提琴不会成为创作音轨。

### 课程材料

课堂不只使用西方经典作品，也包括中国童谣、地方民歌、民族器乐和儿童熟悉的生活音乐。

达尔克罗兹、柯达伊、奥尔夫和铃木只作为设计依据，不做成四个课程入口：

- 达尔克罗兹：身体感受、主动聆听和即兴。
- 柯达伊：哼唱、内在听觉、音乐母语和循序学习。
- 奥尔夫：动作、语言、合作与创编。
- 铃木：重复聆听、模仿和渐进练习。

产品不宣称完整实施其中任何一种体系，也不宣称必然提高智力或学业成绩。

## 3. 儿童音乐创作

### 主流程

> 选择感觉与律动 → 动物编排 → 实时混音 → 可选AI优化 → 生成旋律线与身体演奏

节奏热身移动到教师课堂端；拍手录制降为创作页面里的可选输入，不再是儿童完成作品的前置步骤。

音乐课堂和儿童创作是两个独立入口，但共享“旋律线、身体演奏、四段结构”三种表达方式。课堂可以引导孩子进入创作，儿童作品也可以再次生成全班活动。

### 感觉贴纸

首版提供开心、安静、勇敢和想念四种音乐起点。

感觉贴纸选择提前制作好的旋律材料、调性色彩和视觉氛围，不判断儿童心理状态，也不把某种感觉固定成唯一的性格解释。

### 律动贴纸

- 律动贴纸选择提前制作好的节奏型、速度和重音方式。
- 儿童端使用“稳稳走、蹦蹦跳、摇一摇、向前冲”等身体化名称，不把律动简单等同于音乐流派。
- 感觉和律动共同确定一套音乐素材；两者选定后不在课堂现场生成新声音。
- 节奏热身保留在教师课堂端，不再要求儿童录制拍手。

### 五只动物乐队

| 动物 | 音乐职责 | 身体动作 |
| --- | --- | --- |
| 小狗 | 鼓，承载孩子的拍手节奏 | 拍手 |
| 小兔 | 使用do、re、mi等唱名演唱旋律 | 单手画弧线 |
| 小熊 | 使用键盘演奏主旋律和简单和声 | 双手向下按 |
| 小猫 | 贝斯和低音稳定感 | 轻轻跺脚 |
| 小狮子 | 小号短句、长短音、强弱和呼吸 | 手从嘴边向前送，假装吹奏 |

小熊和小兔使用同一份旋律乐谱：小熊提供稳定的键盘音高参考，小兔按相同音高和节奏演唱唱名。只有小熊时是器乐旋律，只有小兔时是唱名旋律，两者同时出现时形成键盘领奏与唱名演唱。小号只在乐句空隙、重音和结尾出现，与唱名形成问答或留白。

这五只动物是儿童创作的固定配置。其他教学乐器不能加入。

### 小兔唱名模式

一期只提供唱名演唱，不加入“啦啦唱”和歌词演唱。

- 使用do、re、mi、fa、sol、la、si等唱名。
- 小兔唱名和小熊键盘必须来自同一份旋律数据，不能分别自由生成。
- 主产品提供“听小兔唱、和小兔一起唱、轮到我唱”三个课堂状态。
- “轮到我唱”时小兔静音，小熊继续提供键盘音高参考。
- 不录制儿童声音，不判断音准，也不评分。

歌词演唱作为后续功能，需要在唱名模式、儿童跟唱和教师课堂验证稳定后再加入。

### 四段编排

- 整首作品分为四个乐段，每段内部可包含四小节。
- 小狗默认出现在四段中，儿童可以移除。
- 动物可以出现在多个乐段，但同一乐段不能重复。
- 同时支持拖动和“先点动物、再点乐段”。
- 点击单段可试听，“从头听”依次播放四段。
- 编排页上方保持纸偶舞台，下方保持时间轴和贴纸盒。
- 动物只在乐段边界上下场；完整播放后形成乐队合照。

当前版本不加入速度、力度或音乐方法贴纸，避免时间轴过于复杂。

### 实时混音与轻量AI优化

每组“感觉＋律动”对应五条提前制作、长度相同、拍点对齐的动物循环分轨。四个乐段不需要分别制作音频，只记录每只动物在哪些乐段发声。

儿童完成编排后，系统立即生成“我的实时混音”：

- 五条分轨从同一拍点保持同步。
- 系统只在乐段边界打开或关闭相应动物音轨。
- 未选择的动物不发声。
- 原始版本不依赖联网或AI生成，必须能够立即播放。

“AI优化版”是可选的轻量处理，不重新作曲，也不重新生成整首音乐。它只可以：

- 调整音量平衡、均衡、空间感和整体响度。
- 在乐段边界加入淡入淡出。
- 从经过审核的素材库中选择过渡和结束音。
- 降低循环拼接感，让四段连接得更自然。

AI优化不能改变动物出场时间、增加未选择的乐器、替换基础旋律或覆盖儿童的原始版本。优化失败时，实时混音版仍可正常播放、保存和进入课堂。

### AI作品转译

完成编排后，系统根据已知的音轨、拍点和动物出场数据，把儿童作品转化为可理解、可跟随的课堂内容；语言模型只负责生成适龄的提示和教师引导语。

AI可以：

- 显示旋律高低线条和四段结构。
- 根据实际节奏生成拍手、拍腿、停顿等身体演奏。
- 生成适合全班跟随的提示和简短教师引导语。
- 提供作品标题和寄语建议。

AI不能：

- 改变动物出场。
- 删除或替换儿童的拍手节奏。
- 擅自加入儿童没有选择的乐器。
- 把自己的版本标成儿童原创。

儿童可以返回修改作品。AI分析失败时，原始编排仍能正常播放和保存。

### 音乐明信片

最终作品包含标题、情绪背景、动物乐队合照、播放按钮、作者标识和一句寄语。

固定显示：

> 由我选择与编排 · AI提供音乐素材和混音帮助

作品默认只保存在当前设备。分享需要成年人确认，只生成私密、有期限的链接；不设置公开广场、点赞、排行或陌生人评论。

### 一起演

“一起演”是连接创作与课堂的核心体验：

1. 先做一次简短动作示范。
2. 给四拍起拍。
3. 屏幕按动物出场提示动作。
4. 完整表演后共同挥手结束。

设备不录音、不录像、不识别动作，也不评分。

## 4. AI与技术分工

| 工作 | 负责方式 |
| --- | --- |
| 感觉、律动和五只动物的声音 | AI提前生成、人工审核的分轨素材 |
| 四段动物出场与即时播放 | 本地实时混音 |
| 音量、淡入淡出、过渡、结尾和整体响度 | 轻量AI混音与审核素材库 |
| 拍点、响度和音高曲线 | 本地或专业音频算法 |
| 音乐结构、乐器、情绪和教学语言解释 | 语言模型 |
| 旋律线、动作、口令和难度版本 | AI生成候选脚本 |
| 教学安全、音乐准确性和年龄适配 | 音乐教师或产品审核人员最终确认 |

音乐素材采用“提前生成、人工审核、稳定播放”。儿童完成编排后先得到本地实时混音；联网AI优化是可选增强，不阻塞创作和课堂。

## 5. 当前Demo范围

Demo只验证：

> 普通教师能否把一首歌曲快速变成可用的音乐活动；孩子能否在体验之后完成作品；这件作品能否再次变成全班共同参与的课堂内容。

当前实现：

- 教师模式中的一首内置示范歌曲。
- 旋律线、四拍身体演奏和投屏课堂。
- 感觉贴纸与律动贴纸。
- 五只动物、四段时间轴和纸偶舞台。
- 原始实时混音与轻量AI优化对比。
- 儿童作品的旋律线与身体演奏转译。
- 音乐明信片、本地保存和分享确认作为可选支线。
- 无摄像头、无动作评分。

任意歌曲上传、正式课程库、麦克风拍手识别和教学乐器贴纸不进入当前三分钟Demo，作为下一阶段。

完整版音乐素材由4种感觉、4种律动和5只动物组成，共16套音乐组合、80条对齐的循环分轨。四个乐段重复使用同一套分轨，只控制音轨开关；另外准备少量经过审核的过渡和结束素材。参赛Demo可以先完成2种感觉×2种律动×5只动物，共20条分轨。正式素材未完成前，原型继续使用浏览器合成音。

## 6. 三分钟演示

1. **0:00—0:20**：说明非专业教师缺少音乐活动设计能力。
2. **0:20—0:55**：选择一首内置歌曲，展示AI生成的旋律线、身体演奏和教师引导语。
3. **0:55—1:20**：进入投屏课堂，跟随旋律线完成一轮身体演奏。
4. **1:20—1:35**：孩子选择一种感觉，进入贴纸创作。
5. **1:35—2:10**：把动物放入四段时间轴并试听作品。
6. **2:10—2:35**：把儿童作品转化为旋律线和身体演奏。
7. **2:35—3:00**：全班点击“一起演”，共同表演孩子创作的音乐。

演示结束语：

> AI先帮助老师把音乐变得可教，再帮助孩子把感受变成作品，最后让孩子的作品成为全班共同演奏的音乐。
