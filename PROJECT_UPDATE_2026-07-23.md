# 项目更新与续做交接（2026-07-23）

这份文件用于换电脑后快速恢复工作。仓库地址：`https://github.com/Mio0707/music.git`。

## 今天完成了什么

- 完成可点击的“动物音乐明信片”核心流程原型：情绪选择、4 拍热身、4 拍拍手录制、动物乐队编排、AI 润色对比、音乐明信片与“一起演”。
- 节奏教学规则已确认：孩子的热身与拍手录制均为 **4 拍**；当前 Demo 使用四分音符位置，后续才在同一组 4 拍中加入八分音符位置。
- 乐队角色已确认：小狗=鼓/拍手节奏，小兔=无歌词演唱，小熊=键盘主旋律，小猫=贝斯，小狮子=吉他和弦。
- 加入了透明贴画素材：五位动物各有头像和演奏姿态两套图。
- 动物编排页改为“上方常驻纸偶舞台 + 下方四段时间轴 + 贴纸盒”。点击试听或“从头听”时，舞台展示当前乐段的动物。
- 舞台已改为固定背景：新增当前乐段的动物会从下方翻上场；移出当前乐段的动物会向下翻退场，舞台幕布不再整体闪动。

## 明天最快的打开方式

1. 克隆或拉取仓库后，进入项目目录。
2. 直接双击 `prototype/index.html`，即可体验主流程。
3. 如需测试浏览器麦克风权限，在项目目录运行：

   ```powershell
   python -m http.server 4173
   ```

   然后访问 `http://127.0.0.1:4173`。

这个原型没有安装包或构建步骤；`index.html`、`styles.css` 和 `app.js` 就像一本小绘本的封面、画面和互动机关，直接打开即可工作。

## 建议的下一步

1. 在浏览器完整走一遍流程，重点体验“动物编排”页的拖放、试听和舞台上下场是否符合预期。
2. 决定每个乐段的试听音乐时长。现在热身与录制已经是 4 拍，但 `prototype/app.js` 中编排页的试听仍以 8 个音频触发点播放；若希望声音逻辑也严格对应 4 拍，可在下一轮把这一段统一为 4 个触发点或明确为“4 拍内的细分节奏”。
3. 如果视觉方向继续深化，可补充舞台背景、不同情绪的纸质纹理，或为每只动物增加“休息姿态”。
4. 如果准备演示，按 `design/Demo页面与三分钟演示方案.md` 的脚本预先走一次，确认麦克风权限和本地浏览器声音正常。

## 已知的 Demo 边界

- 声音由浏览器实时合成，用于交互验证，不是正式音乐素材。
- “AI 润色”是本地模拟的版本差异，尚未连接线上模型。
- 保存使用浏览器本地存储；分享只展示成年人确认流程，不会上传真实儿童作品。
- 原型不使用摄像头，也不会对孩子动作评分。

## 换电脑后的 Git 操作

首次使用新电脑：

```powershell
git clone https://github.com/Mio0707/music.git
cd music
```

如果已经有这个项目文件夹：

```powershell
git pull origin main
```

开始修改前先执行一次 `git pull origin main`，结束后由 Codex 协助检查、提交和推送即可。
