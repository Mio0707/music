# AI音乐素材生产规范

版本：0.2（MiniMax主流程／Mureka备选流程）  
更新日期：2026-07-24

## 1. 文档目的

本文件独立记录儿童音乐产品中贴纸音频素材的：

- 音乐语言定义。
- AI制作流程。
- 提示词模板。
- 文件和数据规范。
- 质量检查规则。
- 失败处理和版本记录。

本文件不修改产品功能，也不代替 `PRODUCT.md`。当前先作为人工执行和持续调整的生产手册；全部规则稳定后，再整理成可重复运行的Skill。

## 2. 核心生产原则

### 2.1 生产单位

音频不能按照单张贴纸彼此独立生成。正确的生产单位是：

> 一个“感觉贴纸＋律动贴纸”组合 = 一套音乐工程  
> 每套音乐工程再输出5只动物的对齐分轨

完整版及参赛Demo规模：

> 4种感觉 × 4种律动 × 5只动物 = 16套组合、80条动物分轨

参赛Demo直接制作完整素材矩阵，不使用2种感觉×2种律动的缩减方案。

80条分轨不是80次彼此独立的音乐生成，而是16次“组合音乐工程”的批量生产。每套组合先确定统一骨架，再输出5条动物分轨。

### 2.2 统一音乐骨架

同一套组合必须先确定一份统一音乐骨架，再制作动物分轨。统一骨架至少包括：

- 调性和首调唱名关系。
- 拍号。
- 速度。
- 两小节主旋律。
- 和声走向。
- 律动和重音。
- 小节起止位置。
- 循环点。

小熊键盘和小兔唱名必须使用完全相同的主旋律、音高和节奏。其他动物围绕同一调性、速度和小节网格制作。

### 2.3 运行方式

- 每条动物分轨长度相同，并从同一个时间点开始。
- 一个基础分轨覆盖两小节。
- 四个乐段重复使用同一套两小节分轨，只改变动物音轨的开关。
- 四段共八小节。
- 完整体验默认把八小节播放两遍。
- 产品运行时只做实时混音，不临时生成音乐。

### 2.4 80条分轨的批量生产方式

批量生产按感觉分为4组，每组包含4种律动：

| 生产批次 | 感觉 | 需要完成的组合 | 分轨数量 |
| --- | --- | --- | --- |
| A | 开心 | 开心分别搭配4种律动 | 4套、20条 |
| B | 安静 | 安静分别搭配4种律动 | 4套、20条 |
| C | 勇敢 | 勇敢分别搭配4种律动 | 4套、20条 |
| D | 想念 | 想念分别搭配4种律动 | 4套、20条 |

正式批量生成前，先完成其中1套纵向测试组合，验证主旋律、首调唱名、5条分轨、循环和实时混音。测试通过后，再按照同一数据和文件规范生产其余15套。

每个批次必须同时交付：

- 4套统一音乐骨架。
- 20条动物分轨。
- 4条合奏试听版。
- 4份音乐信息记录。
- 4份完整提示词和生成记录。
- 该批次的检查清单。

同一感觉下的4套组合应保持可以辨认的情绪家族特征，但4种律动必须在速度、重音、节奏型和身体感受上形成清楚差异。不同组合不能只更换鼓点而让其他4条分轨完全相同。

## 3. 三类贴纸分别控制什么

### 3.1 感觉贴纸

感觉贴纸主要决定：

- 主旋律的性格和走向。
- 调式色彩。
- 和声明暗。
- 音区。
- 句尾感觉。
- 整体空间感和视觉氛围。

感觉贴纸不单独决定：

- 具体乐器。
- 具体节奏型。
- 动物何时进入。
- 最终编排结构。

首版感觉贴纸：

| 贴纸 | 核心音乐语言 | 建议旋律 | 建议和声与音色 | 避免 |
| --- | --- | --- | --- | --- |
| 开心 | 明亮、轻巧、有向上感 | 短句清楚，适量上行和跳进 | 明亮调性色彩、清晰音区 | 过快、尖锐、持续兴奋 |
| 安静 | 平稳、柔和、有呼吸 | 级进为主，音程变化较小 | 柔和和声、留白、较少高频 | 过度悲伤、拖沓 |
| 勇敢 | 稳定、坚定、向前 | 节奏明确，句尾有落点 | 稳定低音、清楚重拍 | 军事化、攻击性 |
| 想念 | 温柔、稍有距离感 | 较长线条，适量下行或回望感 | 温暖和声、适度空间感 | 过度悲伤、成人化煽情 |

上表是方向定义，不是最终音乐参数。具体调性、音域和和声需要通过试听后锁定。

### 3.2 律动贴纸

律动贴纸主要决定：

- 速度范围。
- 节奏型。
- 重拍位置。
- 音符疏密。
- 行进感、摇摆感或弹跳感。
- 身体演奏的逐拍动作基础。

律动贴纸不使用“爵士、摇滚、电子”等流派名称作为儿童入口。

首版律动贴纸：

| 贴纸 | 核心音乐语言 | 节拍感 | 建议身体感受 | 避免 |
| --- | --- | --- | --- | --- |
| 稳稳走 | 均匀、可靠、容易跟随 | 每拍清楚，重拍稳定 | 走、拍腿、拍手 | 过多切分和突然停顿 |
| 蹦蹦跳 | 轻快、有弹性 | 短音较多，重拍有弹跳感 | 拍手、轻点、交替动作 | 速度过快、动作来不及 |
| 摇一摇 | 左右摆动、柔和循环 | 重拍不过分生硬 | 左右交替、拍腿 | 复杂摇摆比例 |
| 向前冲 | 推进、连续、有方向 | 重拍明确，密度稍高 | 拍桌、拍手、向前动作 | 紧张、压迫或攻击感 |

每种律动的准确速度、拍号和重音模板需要在正式生产前锁定。

### 3.3 动物贴纸

动物贴纸决定由哪种声音加入实时混音：

| 动物 | 固定音乐职责 | 与其他分轨的关系 | 禁止 |
| --- | --- | --- | --- |
| 小狗 | 鼓和预设节奏型 | 体现律动贴纸的速度、拍点和重音 | 自由改变速度或加入复杂独奏 |
| 小熊 | 键盘主旋律和简单和声 | 提供稳定音高参考 | 改写主旋律或遮盖其他声音 |
| 小兔 | 使用首调唱名演唱主旋律 | 与小熊使用同一份旋律数据 | 自由改词、自由转调、与小熊不同步 |
| 小猫 | 贝斯和低音稳定感 | 跟随和声根音和律动 | 复杂炫技或抢占主旋律 |
| 小狮子 | 小号重音、问答、过渡和结束短句 | 在旋律空隙和重点位置出现 | 从头到尾持续演奏或与唱名冲突 |

## 4. 单套音乐的标准制作流程

### 第一步：填写组合任务卡

每次生成前先填写：

- 感觉贴纸：
- 律动贴纸：
- 拍号：
- 速度：
- 调性：
- 两小节时长：
- 目标年龄：
- 主旋律音域：
- 预期身体感受：
- 禁止出现的特征：

没有填写完整任务卡时，不进入批量生成。

### 第二步：生成统一音乐骨架

先用AI生成或辅助生成：

- 两小节主旋律。
- 和声框架。
- 律动框架。
- 结构化乐谱或MIDI。
- 首调唱名序列。

先审核骨架，再制作动物声音。不能先分别生成5条音频，再尝试强行拼合。

### 第三步：制作小熊和小兔

1. 根据主旋律制作小熊键盘。
2. 根据同一主旋律生成小兔首调唱名。
3. 检查每一个音的开始位置、持续时间和音高。
4. 两者同时播放时，不能出现明显错音、抢拍或拖拍。

### 第四步：制作小狗和小猫

1. 小狗根据律动贴纸生成鼓和拍点。
2. 小猫根据和声根音及律动生成贝斯。
3. 先检查小狗和小猫是否形成稳定基础。
4. 再加入小熊检查低音是否与和声冲突。

### 第五步：制作小狮子

小狮子最后制作，只能使用：

- 句尾回应。
- 旋律空隙。
- 重音强调。
- 过渡短句。
- 结尾音。

如果小狮子加入后影响小兔唱名的清晰度，优先减少小狮子，而不是提高小兔音量。

### 第六步：统一导出

每套组合至少导出：

- 小狗分轨。
- 小熊分轨。
- 小兔分轨。
- 小猫分轨。
- 小狮子分轨。
- 五只动物合奏试听版。
- 主旋律乐谱或MIDI。
- 音乐信息文件。

### 第七步：组合测试

至少试听：

- 每只动物单独播放。
- 小狗＋小猫。
- 小熊＋小兔。
- 小狗＋小猫＋小熊。
- 五只动物全部播放。
- 任意一只动物中途加入和退出。
- 两小节连续循环4次。
- 八小节整体连续播放2遍。

## 5. AI提示词结构

提示词不能只写“生成一首开心的儿童音乐”。每次提示必须包含音乐任务、固定参数、输出要求和禁止项。

### 5.1 统一骨架提示词模板

```text
为6—12岁儿童音乐创作产品设计一套可循环的两小节音乐骨架。

感觉贴纸：{感觉名称}
感觉语言：{旋律、和声、音区和空间感描述}
律动贴纸：{律动名称}
律动语言：{速度、拍号、节奏密度和重音描述}
速度：{BPM}
拍号：{拍号}
调性：{调性}
长度：严格两小节

请先输出一条简单、容易哼唱的主旋律，再输出简单和声与低音框架。
旋律需要适合转换为首调唱名，音域适合儿童跟唱。
音乐必须能够无缝循环4次，不要前奏，不要长尾，不要临时变速，不要转调。
避免复杂切分、炫技、成人化情绪、过度戏剧化和过密编曲。

输出：
1. 主旋律音符与节奏；
2. 首调唱名；
3. 和声；
4. 低音根音；
5. 拍点与重音；
6. 可导出的乐谱或MIDI数据。
```

### 5.2 小狗提示词模板

```text
根据已经确认的两小节音乐骨架，制作“小狗”鼓和节奏分轨。

严格使用：
- 速度：{BPM}
- 拍号：{拍号}
- 长度：两小节
- 律动：{律动语言}

要求每拍清楚、儿童容易跟随，突出规定的重拍。
不得改变速度、不得增加前奏、不得加入旋律、不得产生无法循环的长尾。
从第1拍准确开始，并与其他分轨保持完全相同的长度。
```

### 5.3 小熊提示词模板

```text
根据给定主旋律制作“小熊”键盘分轨。

主旋律：{音符与节奏}
和声：{和声框架}
速度：{BPM}
拍号：{拍号}
长度：两小节

键盘需要清楚演奏主旋律，并只加入简单、轻量的和声。
主旋律的每个音高和节奏必须与提供的数据一致。
不得即兴改写旋律，不得增加复杂装饰，不得转调，不得改变长度。
```

### 5.4 小兔提示词模板

```text
根据给定主旋律制作“小兔”首调唱名分轨。

主旋律：{音符与节奏}
首调唱名：{do re mi序列}
速度：{BPM}
拍号：{拍号}
长度：两小节

使用清楚、温和、适合儿童模仿的声音演唱唱名。
每个唱名的音高、开始位置和持续时间必须与主旋律一致。
不要添加歌词、啦啦唱、转音、和声演唱、自由节奏或额外停顿。
```

### 5.5 小猫提示词模板

```text
根据给定和声与律动制作“小猫”贝斯分轨。

和声根音：{根音序列}
律动：{律动语言}
速度：{BPM}
拍号：{拍号}
长度：两小节

低音需要稳定、简单、温暖，为儿童提供清楚的拍点和和声基础。
不要演奏主旋律，不要炫技，不要使用过密音符，不得改变和声或循环长度。
```

### 5.6 小狮子提示词模板

```text
根据给定主旋律与空隙位置制作“小狮子”小号分轨。

允许出现的位置：{重音、空隙、过渡、结尾位置}
速度：{BPM}
拍号：{拍号}
长度：两小节

只使用少量短句、重音或回应，不要持续演奏。
不得遮盖主旋律和唱名，不得改写和声，不得在未允许的位置发声。
音色明亮但不能刺耳，结尾需要便于循环。
```

### 5.7 通用禁止词

所有提示词根据工具能力补充以下限制：

```text
不要前奏，不要人声歌词，不要环境对白，不要掌声，不要转调，
不要临时变速，不要复杂切分，不要过密编曲，不要长混响尾音，
不要在循环边界增加结束式，不要改变指定小节数。
```

## 6. 音频与文件规范

以下为首版建议值，正式批量生产前需要试听确认：

- 母版优先保存为WAV。
- 建议采样率48 kHz。
- 建议位深24-bit。
- 同一套组合所有分轨声道格式一致。
- 所有分轨必须完全等长。
- 所有分轨开头保留方式一致，不能有不同长度的静音。
- 循环版末尾不能保留破坏循环的长混响。
- 网页使用时另行导出体积较小的播放版本。
- 母版和网页版本分别保存，不能只保留压缩音频。

### 文件命名

建议格式：

```text
{感觉}_{律动}_{动物}_{版本}.{格式}
```

示例：

```text
happy_bounce_dog_v01.wav
happy_bounce_bear_v01.wav
happy_bounce_rabbit_v01.wav
happy_bounce_cat_v01.wav
happy_bounce_lion_v01.wav
happy_bounce_preview_v01.wav
```

同一套组合必须使用相同版本号。修改主旋律、速度、拍号或长度时，整套组合统一升级版本。

正式数据还需要记录：

- 首调唱名。

- 完整提示词。
- 生成日期。
- 审核结果和修改说明。

## 8. 质量检查清单

### 音乐正确性

- [ ] 五条分轨使用同一速度、拍号和调性。
- [ ] 小熊与小兔旋律完全一致。
- [ ] 小猫低音与和声一致。
- [ ] 小狗重拍符合律动定义。
- [ ] 小狮子没有遮盖主旋律。
- [ ] 没有未计划的转调或速度变化。

### 循环和同步

- [ ] 五条分轨长度完全一致。
- [ ] 所有分轨从同一个时间点开始。
- [ ] 两小节循环4次没有明显断点。
- [ ] 动物在乐段边界加入时不抢拍。
- [ ] 动物退出时没有破坏下一小节的长尾。
- [ ] 八小节播放两遍仍然自然。

### 儿童适配

- [ ] 主旋律容易记忆和哼唱。
- [ ] 音域适合儿童。
- [ ] 速度不会让身体动作跟不上。
- [ ] 情绪表达清楚但不过度。
- [ ] 没有刺耳、惊吓或攻击性声音。
- [ ] 不依赖成人音乐流派知识才能理解。

### 产品混音

- [ ] 单独一只动物播放时仍然成立。
- [ ] 任意两只或三只动物组合不会严重冲突。
- [ ] 五只动物一起播放不过载。
- [ ] 小兔唱名始终清楚。
- [ ] 小熊可以在小兔静音时独立提供音高参考。

## 9. 失败处理

- 分轨无法同步：回到统一乐谱或MIDI重新生成，不手工拖动到“差不多”。
- 小熊与小兔不同：以最终确认的主旋律数据为准，重新生成其中一条。
- 循环有断点：缩短尾音或重新设计结尾，不使用大段交叉淡化掩盖错误。
- 任意组合冲突：优先简化动物分轨，不依赖最终混音强行修复。
- AI多次生成仍不稳定：保留AI生成的乐谱或MIDI，改用稳定音源渲染。
- 素材未通过审核：不得放入正式Demo，继续使用浏览器备用音源。

```text
- 循环：
- 旋律一致：
- 任意组合：
- 儿童适配：

发现的问题：
修改方式：
最终结论：通过 / 修改后复审 / 淘汰
```

## 11. 当前待讨论内容

以下参数尚未最终锁定：

- 每种律动的准确BPM和拍号。
- 每种感觉的准确调性、和声和主旋律音域。
- 小兔声音的年龄感、音色和发音方式。
- 最终使用哪种AI音乐生成、歌声生成和音频处理工具。
- 是否能够直接获得结构化乐谱、MIDI和真实分轨。
- 网页播放版本使用哪种压缩格式。
- 音量和响度的最终统一标准。
- 小狮子的结尾音是否单独制作，不放在循环分轨中。

这些内容需要通过第一套测试组合确定，再开始批量生成80条正式分轨。

## 12. 未来整理为Skill

当前阶段不创建Skill。规则稳定后，Skill至少需要包含：

- 输入：感觉、律动、速度、拍号、调性和版本。
- 读取：感觉语言、律动语言、动物职责和禁止项。
- 生成：统一骨架、5条动物提示词、文件命名和信息记录。
- 检查：同步、循环、旋律一致、儿童适配和组合试听。
- 输出：音频包、提示词、音乐数据、审核报告和版本记录。
- 中止条件：缺少统一骨架、分轨不同步、小熊与小兔旋律不一致或素材未通过审核。

只有当至少一套完整组合经过真实生产、修改和验收后，才开始把流程封装成Skill，避免把未经验证的假设固化。

## 13. 当前工具路线（2026-07-24确认）

这一期优先使用国内能够注册、充值和正式调用的模型，避免海外服务的支付和网络问题。

### 13.1 主路线

> MiniMax国内音乐API负责生成音乐方向和候选成品  
> 结构化乐谱、MIDI、固定网格和自动检查负责保证准确  
> 最终80条动物分轨必须经过人工审核

MiniMax国内平台当前使用：

- 国内文档：`https://platform.minimaxi.com/docs/`
- 国内接口：`https://api.minimaxi.com`
- 当前先测试的音乐模型：`music-2.6`
- 主要能力：文字生成音乐、纯器乐、参考音频翻唱、歌词歌曲。
- 当前公开接口未提供：直接输出MIDI、指定输出5条动物分轨、用参数严格规定2小节。

因此，MiniMax不能直接被当作“输入一句话，稳定返回80条最终分轨”的工具。它在本流程中的主要职责是：

1. 为16种“感觉＋律动”组合产生可试听的音乐方向。
2. 提供候选旋律、和声、律动和配器灵感。
3. 为选中的音乐方向生成较完整的参考混音。
4. 后续生成古诗演唱示例或儿童作品的完整AI优化版本。
5. 在参考音频翻唱可控性通过测试后，尝试制作特定音色候选。

### 13.2 备选路线

Mureka作为第一备选，国内平台为：

- 国内文档：`https://platform.mureka.cn/docs/`
- 主要能力：器乐生成、MIDI旋律参考、补充单独乐器、音乐转录和分轨下载。

如果MiniMax在旋律保持、短片段生成或分轨生产方面无法达到标准，就把已经确认的MIDI交给Mureka继续制作。Mureka也必须经过同样的两小节、同步、循环和儿童适配检查，不能因为支持MIDI或分轨就直接视为合格。

### 13.3 暂不进入主流程

- Suno、Eleven Music、Google Lyria等海外服务只作为效果参考，不进入交付依赖。
- 阿里云Fun-Music只作为阿里云内部的参考候选，不承担标准MIDI和最终分轨。
- ACE-Step等开源模型作为后期自部署备选，本期不阻塞80条素材交付。

## 14. MiniMax主路线：单套组合完整制作流程

以下流程先用于“开心＋蹦蹦跳”纵向测试。通过后，再复制到其余15套组合。

### 14.1 阶段A：建立组合任务卡

先填写第4节中的任务卡，并额外记录：

- MiniMax模型版本。
- 预计生成次数。
- 是否生成纯器乐。
- 是否提供参考音频。
- 希望保留的核心动机。
- 计划从成品中截取哪一段。

第一套测试组合暂用以下参数验证流程，不代表最终锁定：

```text
感觉：开心
律动：蹦蹦跳
拍号：4/4
速度：96 BPM
调性：C大调
基础循环：2小节
两小节理论时长：5秒
目标年龄：6—12岁
主旋律音域：C4—C5附近
```

### 14.2 阶段B：先生成结构化音乐骨架

在调用MiniMax之前，先用通义千问或人工规则产生一份候选骨架：

```text
感觉＋律动
→ 主旋律音符与时值
→ 首调唱名
→ 和弦
→ 贝斯根音
→ 鼓点网格
→ 小狮子允许出现的位置
→ 标准MIDI
```

这份骨架是最终准确性的依据。MiniMax生成结果只能提出新的音乐方向，不能覆盖已经审核的音高、节拍和动物职责。

如果MiniMax给出了更好的旋律，必须先把旋律转录并人工修正为新的标准MIDI，再统一更新小熊、小兔和其他动物，不能只替换某一条音频。

### 14.3 阶段C：用MiniMax生成组合参考曲

先使用纯器乐模式生成完整参考曲。提示词需要突出：

- 儿童音乐。
- 感觉和律动。
- BPM、调性和4/4拍。
- 明确的两小节核心动机。
- 键盘主旋律、鼓、贝斯和少量小号回应。
- 不要前奏、不要转调、不要复杂独奏。
- 核心动机应可以截取并循环。

第一套测试提示词：

```text
为6—12岁儿童设计一段明亮、轻巧、容易跟随的器乐音乐。

感觉是开心，律动是蹦蹦跳。4/4拍，约96 BPM，C大调。
使用清楚、容易哼唱的键盘或木琴主旋律，轻快但不过快的鼓点，
温暖简单的贝斯，以及只在旋律空隙出现的少量明亮小号回应。

音乐围绕一个非常清楚的两小节核心动机展开。
核心动机短句明确，有适量上行和弹跳感，适合儿童用do、re、mi等唱名跟唱。
不要前奏，不要歌词，不要人声，不要转调，不要临时变速，
不要复杂切分，不要炫技，不要密集编曲，不要长混响尾音。
让核心动机的首尾容易剪辑成自然循环。
```

每次任务至少生成2个候选，最多保留3个进入下一步。没有候选通过基础试听时，先修改提示词，不立即批量生成。

### 14.4 阶段D：选择和截取候选段落

从MiniMax完整结果中寻找：

- 节奏稳定。
- 调性清楚。
- 乐器相对清楚。
- 没有转调。
- 没有明显填充或结束式。
- 连续两小节可以自然重复。

截取时必须先检测真实BPM和拍点，不能只凭波形目测。截取结果记录：

- 原始文件名。
- 截取开始时间。
- 截取结束时间。
- 检测BPM。
- 检测调性。
- 是否需要变速或移调。
- 选择理由。

如果找不到可以自然截取的两小节，MiniMax结果只保留为风格参考，不强行进入最终素材。

### 14.5 阶段E：转成标准乐谱和MIDI

对选中的两小节进行旋律和节拍转录：

1. 自动识别拍点、速度和小节边界。
2. 自动识别候选主旋律。
3. 把音符吸附到规定节拍网格。
4. 限制在儿童适合的音域内。
5. 修正调外音、碎音和错误延音。
6. 人工试听确认。
7. 导出标准MIDI和结构化音符数据。

自动转录只产生候选，最终MIDI必须人工确认。小熊和小兔之后都读取这同一份最终MIDI。

### 14.6 阶段F：制作5条动物分轨

#### 小熊

- 使用最终MIDI渲染键盘主旋律和轻量和声。
- 优先保证旋律清楚，不追求复杂音色。
- 如果尝试MiniMax参考音频翻唱，必须与标准MIDI逐音比较；不通过时使用稳定音源渲染版。

#### 小兔

- 从同一MIDI生成首调唱名序列。
- 第一阶段使用可控的唱名合成或审核后的唱名音节素材。
- MiniMax自由演唱只作为候选，不能作为准确唱名的唯一来源。

#### 小狗

- 根据律动贴纸的固定节奏网格制作鼓点。
- 可以参考MiniMax的鼓点感觉，但最终拍点由网格决定。
- 不从混合音乐中直接截取带有其他乐器残留的鼓声作为正式分轨。

#### 小猫

- 根据和弦根音、律动和标准MIDI制作简单贝斯。
- 可以参考MiniMax贝斯的音色和节奏密度。
- 不允许产生与主旋律争抢的复杂低音。

#### 小狮子

- 根据任务卡中允许出现的位置制作少量小号短句。
- 可以参考MiniMax小号的音色和回应方式。
- 如果无法干净分离小号，使用稳定音源按标准MIDI重新渲染。

### 14.7 阶段G：统一音频

把5条分轨统一处理为：

- 相同采样率、位深和声道格式。
- 相同起点。
- 相同结束点。
- 严格2小节。
- 不带破坏循环的长尾。
- 留有足够动态空间，五轨合奏不能削波。

同时导出：

- 5条WAV母版。
- 1条五轨合奏试听版。
- 1份标准MIDI。
- 1份结构化音乐信息。
- 1份生产记录。

### 14.8 阶段H：测试和结论

执行第8节全部检查。测试结果只能是：

- `通过`：进入正式素材库。
- `修改后复审`：保留中间文件，修正后再检查。
- `只作参考`：MiniMax结果提供方向，但最终不用该音频。
- `淘汰`：删除素材引用，但保留失败原因记录。

## 15. MiniMax参考音频翻唱的试验规则

MiniMax参考音频翻唱只在以下情况下试验：

1. 已经有准确的标准MIDI。
2. 已经把MIDI渲染成清楚的参考音频。
3. 参考音频达到接口要求的最短时长。
4. 已经有不用AI翻唱也能工作的保底分轨。

如果两小节不足接口要求的6秒，可以把两小节准确重复一次，形成四小节参考音频，再生成候选；最终仍只选取和检查两小节。不能通过拉长静音伪造参考时长。

翻唱测试只判断：

- 是否保持核心旋律。
- 是否保持速度和拍点。
- 是否产生额外伴奏。
- 是否增加前奏、尾奏或转调。
- 是否可以截取出干净的目标乐器。

任一关键条件不满足，就停止使用该结果制作正式分轨。

## 16. Mureka备选路线

当MiniMax出现以下情况时切换Mureka：

- 多次生成仍没有清楚的两小节核心动机。
- 无法保持指定旋律。
- 无法获得可用的单独乐器。
- 生成长度和结构难以整理。
- 完整混音很好听，但无法转化成5条可复用分轨。

### 16.1 Mureka输入

优先输入已经审核的：

- 标准MIDI。
- 组合任务卡。
- 感觉与律动提示词。
- 目标乐器。
- 参考混音或参考音频。

### 16.2 Mureka执行顺序

```text
上传标准MIDI作为melody
→ 先生成完整器乐候选
→ 再尝试生成互补的单独乐器
→ 下载可用分轨
→ 裁剪并对齐到标准两小节
→ 与标准MIDI逐音、逐拍比较
→ 通过后进入统一音频处理
```

建议优先测试小熊键盘，再测试小狗鼓、小猫贝斯和小狮子小号。小兔唱名仍使用专门的可控唱名流程。

### 16.3 Mureka失败时

- 旋律改变：回到标准MIDI，用稳定音源渲染。
- 分轨有残留：不用于正式混音，改为音色参考。
- 长度不准确：只在音乐内容完整时按拍点裁剪；内容本身不完整则淘汰。
- 多次生成不一致：锁定已通过的结果，不继续追求随机的“更好版本”。

## 17. 阿里云在生产流程中的职责

阿里云是整个生产流程的控制和存储平台，不要求所有音乐都由阿里模型生成。

| 阿里云模块 | 负责内容 |
| --- | --- |
| 百炼／通义千问 | 组合任务卡、结构化乐谱候选、提示词整理、失败原因归纳 |
| 工作流／CloudFlow | 依次调用MiniMax或Mureka、查询任务、失败重试、人工审核节点 |
| 函数计算 | MIDI生成、节拍网格、文件检查、裁剪、统一命名和打包 |
| OSS | 保存原始结果、MIDI、母版、网页版、报告和历史版本 |
| 数据表 | 保存16套组合的状态、模型版本、成本、审核结论 |
| PAI-EAS（后期） | 部署ACE-Step、唱名合成或其他本地开源模型 |

正式流程必须能够在MiniMax或Mureka暂时不可用时继续：

> 标准MIDI＋稳定音源渲染是保底路线  
> 外部音乐模型提高创意和音色质量，但不能成为唯一生路

### 18.2 再按感觉分4批

```text
A批：开心 × 4种律动 = 20条
B批：安静 × 4种律动 = 20条
C批：勇敢 × 4种律动 = 20条
D批：想念 × 4种律动 = 20条
```

每批流程：

1. 先审核4份音乐骨架。
2. 再生成4份MiniMax参考曲。
3. 选择并确认4套标准MIDI。
4. 按统一顺序制作20条分轨。
5. 完成4条合奏试听版。
6. 整批执行自动检查和人工试听。
7. 通过后再进入下一批。

### 18.3 批量生成限制

- 不一次性提交80条互不关联的提示词。
- 不在骨架未确认时批量生成5只动物。
- 不把同一感觉下4种律动只做成“换鼓点”。
- 不因为已经付费生成就降低审核标准。
- 不覆盖已经通过的版本；新结果必须使用新版本号。
